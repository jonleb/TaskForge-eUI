The eui-tree component supports virtual scrolling to improve performance when working with large datasets. This behavior can be customized through the following options:

- virtualScrollThreshold (default: 800) – the minimum number of nodes required before virtual scrolling is enabled.
- virtualScrollPageSize (default: 400) – the number of nodes rendered per page when virtual scrolling is active.

Enabling and tuning these options is especially useful when handling a large number of nodes, as it ensures smooth rendering and efficient navigation.
Combined with the built-in search functionality, virtual scrolling provides a faster and more responsive experience for end users.

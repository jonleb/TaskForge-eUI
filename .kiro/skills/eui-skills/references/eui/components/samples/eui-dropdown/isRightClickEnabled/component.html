Usability notice

The eui-dropdown with the isRightClickEnabled option allows to trigger a menu using the right mouse click event instead of the regular left click. This behaviour is known as "contextual menu" which usually lacks the visual trigger.

Contextual menus offers a limited set of choices that are available in the current state or context of the application to which they belongs. Usually the available choices are quick actions menu items related to the selected/targeted element.

Also note that contextual menus have received some criticism from usability analysts when improperly used, as some applications make certain features only available in context menus, which may confuse even experienced users.
So please, be aware that the usage of the contextual menus should be very limited to some special use cases.


UX recommendations

Contextual menus without any visual trigger displayed on the GUI should be announced and documented properly to the users. Usually, the contextual menu appears below the position of the pointer or near the focused element at the cursor position.

We do NOT recommend to trigger it on the full page in order to not interfere with the browser's default context menu which might provide additional functionalities configured by your organization.
Regarding tables, when using the contextual menu for data row actions (see example below), its usage should be clearly informed to the user on how to proceed.
We recommend to use the "action column" instead. See Design pattern sample here.


Important accessibility notice

Please note that contextual menus are NOT accessible by nature and thus, not announced by screen readers (see UX recommendations hereup).

This feature requires the import of EuiTemplateDirective
import { EuiTemplateDirective } from '@eui/components/directives';
Few parts of eui-file-upload can be customized.
  • uploadedFile: Allows to customize the rendering of each file added or defined as existing values. A variable is sent to the template with file informations.
  • browseButton: Allows to customize the rendering of the browse button. (openBrowseWindow method needs to be called)
  • resetButton: Allows to customize the rendering of the reset button. (resetList method needs to be called)
  • progressBar: Allows to customize the rendering of the progress bar. A variable is sent to the template with progress level.

This sample shows how to format the size of the file with the pipe filesize provided by the component.

The removal of a file from the list has to be handled in two ways as a file can be a JS File object (when choosen by a user) or instance of EuiUploadedFileInterface (when provided as existing value). JS File object needs to be removed with the public method removeFromList, instances of EuiUploadedFileInterface needs to be removed with the public method removeFromUploadedList. The type of file can be known thanks to the boolean prop isFileObject in the return object of the template.

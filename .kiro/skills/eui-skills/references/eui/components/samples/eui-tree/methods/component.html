expandAt
It accepts the path:string of the targeted node in tree-data.
The indexes of children items will be interpreted as index of multi-dimensional-array for expand feature.
"." is the separator for path syntax. expandAt('IndexA.IndexB.IndexC....IndexN')
f.e: If you target 2nd item on treeData: expandAt('1')
f.e: If you target 1st item's 1st child: expandAt('0.0')
f.e: If you target 3rd item's 2nd child's 1st child:'s expandAt('2.1.0')

Template-Driven forms allows a "two-way data binding" of ngModel mechanism (like in AngularJs) to transparently keep in sync a form with a view model. Template-driven forms let direct access modify data in your template, but are less explicit than reactive forms because they rely on directives embedded in the template, along with mutable data to track changes asynchronously.

Template-driven forms focus on simple scenarios and are not as reusable and scalable as Reactive forms. They abstract away the underlying form API, and use asynchronous data flow between the view and the data model. The abstraction of template-driven forms also affects testing. Tests are deeply reliant on manual change detection execution to run properly, and require more setup.

Structure of a template driven form :

    <form #myForm="ngForm" id="my-form" (ngSubmit)="onSubmitTemplate(myForm)">
        ...
        ...
    </form>
    <button euiButton form="my-form" type="submit" [disabled]="!myForm.valid">Submit</button>

For more information see Building a template-driven form

## eUI design system evolution

eUI 21 includes the phase one of the new Europa Design System implementation, which was started in collaboration with DIGIT.A.2 UX Office and DG COMM ECL team. In this first phase, we've reworked the design tokens and the core foundations, that now eUI 21 and upcoming ECL version 5 are sharing.


### Unified color palettes

eUI and ECL now are using identifcal raw color palettes and inherited as shown here : 

![v21 europa design system to ecl and eui21 color palettes](assets/images/eui-color-context-theme.png)


### Unified typography for EC

New default EC font has been also integrated : <strong>Inter</strong>, an open source font, with a modern look, international and available in multiple font-weights, more info : https://fonts.google.com/specimen/Inter/about

Although for a font scaling point of view, the default font size L/M/S/XS are offset compared to ECL v5 specs, <strong>M (default font size)</strong>  for eUI corresponds to default browser font-size base of 16px = 1rem, ECL v5 raised it up to 18px for website / content driven.

More info here regarding eUI typography scaling: https://eui.ecdevops.eu/eui-showcase-ux-components-21.x/showcase-design-system/design-tokens/typography


### Unified design tokens (others)

Also other design tokens have been aligned in this eUI 21 versions : border-radius - shadows - spacings, for spacings introducing intermediate sizes to cover a larger spectrum and have more flexibility when designing components.


### Iconography : Phosphor as open-source library

We also wanted to unify in a single icons library to avoid visual inconsistencies and a mixup of different icons coming from different libraries, <strong>Phosphor</strong> an open-source library have been chosen for this purpose, it contains more than 1500 icons that are covering a majority of topics, more info : https://phosphoricons.com/

Note that all eUI default sets have been also updated with Phosphor usage, you can find the migration list of icons here from v19 icon names to the new v21 corresponding sets (either eUI default set / ECL set / Phosphor set).

Migration from eUI 19 to eUI 21 icons : https://eui.ecdevops.eu/eui-showcase-ux-components-21.x/style-guide/components/eui-icon-svg#migration

It's not recommended to use any other icons libraries anymore (FontAwesome which have been provided as transition in eUI 19 is now not provided in eUI assets distribution).


### What has changed from a developer point of view ?

#### Components breaking changes

The major breaking change of this release concerns the standalone components only usage / import for both eUI and eUI ECL components, there's no more exposure of component modules anymore for both libraries.

Refer to the migration guide for more info about other breaking changes.


#### Styles

On the exposed eUI styling ( mainly utility classes and css variables) the new core foundations adoptions are reflected mainly on colors exposure, where only combinations of accessible colors from the default palettes are now only to be used.

## eUI Accessibility(a11y) Automated Testing

In this version we introduced automated accessibility testing by integrating the Axe-core engine into our automated testing framework.

This allows us to catch common accessibility issues during the development process before releasing a new version of eUI and thus ensure that our components meet the EN 301 549 standard criteria.

As described in our [a11y testing tools section](https://eui.ecdevops.eu/eui-showcase-ux-components-21.x/showcase-dev-guide/docs/02-accessibility/01-a11y-testing-tools), automated testing is great, but it can only detect 30%-50% of accessibility issues. Therefore, manual testing is always necessary for a detailed accessibility check. In eUI, we always perform manual testing upon receiving an accessibility request/bug and we also perform random a11y manual checks throughout the year on the same major eUI version.

### Configuration

The tests are split into 4 sections:
- Violations: true (Accessibility issues that need to be fixed)
- Incomplete: true (Tests that need manual review)
- Passes: false (Tests that passed)
- Inapplicable: false (Tests that did not apply to the page)

Attach reports:
- HTML: true (html report enabled by default)
- JSON: false (JSON report disabled by default)

Accessibility failure thresholds for violations:
- Critical: 0 (No critical violations allowed)
- Serious: 1, (Allow 1 serious violation)
- Moderate: 5, (Allow up to 5 moderate violations)
- Minor: 10 (Allow up to 10 minor violations)

Failure thresholds for incomplete tests:
- Critical: 1 (One critical incomplete test allowed)
- Serious: 2, (Allow 2 serious incomplete tests)

### Test results for eUI-21 (09/12/2025)

In eUI 21 we managed to fix all the eUI related a11y issues reported by the automated tests and reached to a pass percentage of 98.76%. The only failing test that still produces a11y errors is coming from the eui-editor since there are errors related to the Quill library used internally that cannot be fixed on our side.

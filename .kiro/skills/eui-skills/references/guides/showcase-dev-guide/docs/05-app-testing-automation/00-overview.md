# Testing - references

As an entry documentation, refer to angular.io website about testing : https://angular.io/guide/testing

## Sample app provided spec files
In the sample app, analyze the provided spec files, they do provide some testing implementation to serve as a basis for creating your own unit testing over components and services.

## CI testing
It's important to mention here that Unit testing using Chrome Headless configuration will not run as default in Bamboo plans
on default Bamboo agents, please contact support for deeper information about CI testing.

## E2E eUI automation framework

B.3 QA team is maintaining the eUI automation framework, build on top of eUI it provides set of tooling and component objects to easily automate your E2E tests on eUI based applications : 

WebdriveIO documentation used in the framework : https://citnet.tech.ec.europa.eu/CITnet/confluence/display/COMPCORP/WDIO+Documentation

package sources : https://citnet.tech.ec.europa.eu/CITnet/stash/projects/CSDR/repos/eui-automation/browse/components?at=refs%2Fheads%2Ffeature%2FCCQA-2022-async

Examples of integration on MyWorkplace : https://citnet.tech.ec.europa.eu/CITnet/stash/projects/CSDR/repos/app-my-workplace-e2e-wd/browse?at=feature%2FCCQA-2022-async

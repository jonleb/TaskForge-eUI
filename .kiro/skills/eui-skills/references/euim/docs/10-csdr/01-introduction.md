# CSDR

## What is it ?

Lot of talks but not easy to describe or with just a few words : a massive helper for developers

I stands for <em class="info">Corporate Shared Development Repositories</em>, and it's main goal is to federate how a development
community should work in collaboration.

Some principles : 
- everyone known by it, knows about everyone's work, it's a small enterprise github if you'd like
- but more than a source code repository it's also a builder / a tester / a server / an helper for development daily tasks.
- it uses an opinionated view over development within huge teams
- it ensures quality and security from a central point of view
- it ensures code quality by enforcings code styles and linting rules, those rules globally maintained by the core team
- it provides the same experience for developers as when it runs within a CI engine
- it enforces centralized pipeline as code using the same tooling as when developing it
- it supports modern tools like slack / gitLab / gitHub
- it takes care of a release process through tooling and configuration files
- it acts as a monorepo when installed localy but behave as a multi-repo for its storage
- it provides simple installation steps with mounting operations : developers install what they want to develop on but uses same tools as others
- it regulates the access rights by security groups maintained by support for initialization and first time use configuration
- it's has a repeatable process that has been battle tested with ~100 devs since 2 years
- it's easy to use !
